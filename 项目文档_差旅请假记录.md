## 一、项目目标与定位

- **一句话**：一个记录全年请假与差旅的纯前端单页应用，主打「玻璃拟态 / 液态拟态」UI，支持跨设备通过分享网址同步数据。
- **技术栈**：原生 HTML + 内联 CSS + 原生 JS，零依赖（仅 CDN 引入 `lz-string` 用于压缩同步链接）。
- **交付物**：单一 `index.html`，双击即可打开，也可挂静态服务器访问。
- **视觉基调**：极浅灰白主色、深黑 `#111827` 强调色、低饱和度状态色、毛玻璃模糊 + 虚实对比。

## 二、核心数据结构（重点，改数据逻辑前必读）

保存在 `localStorage` 键 `travelLeaveRecords`，结构：
```json
{
  "travel": [ { /* 差旅记录 */ } ],
  "leave":  [ { /* 请假记录 */ } ]
}
```

### 差旅记录 travel[] 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，格式 `tr_<时间戳>` |
| `person` | string | 出差人（必填） |
| `reason` | string | 差旅事由（必填） |
| `location` | string | 地点（选填） |
| `amount` | number | 金额 |
| `startDate` | string | 开始日期 `YYYY-MM-DD` |
| `endDate` | string | 结束日期 `YYYY-MM-DD`（单日时等于 startDate） |
| `reimbursed` | boolean | 是否已报销 |
| `note` | string | 备注 |
| `createdAt` | number | 创建时间戳 |

### 请假记录 leave[] 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | `lv_<时间戳>` |
| `type` | string | 事假/病假/年假/调休/其他 |
| `startDate` | string | 开始日期 |
| `endDate` | string | 结束日期 |
| `note` | string | 备注 |
| `createdAt` | number | 创建时间戳 |

### 兼容旧数据
- 旧版单日期字段 `date` 在 `normalizeRecord(record, type)` 中自动转换为 `startDate`/`endDate`（两者都等于 `date`），保证导入旧 JSON 不报错。
- 任何写入 localStorage 前都应经过 `normalizeRecord` 以保证字段完整。

## 三、页面结构（两个主视图 + 多个 Sheet 弹窗）

### 主视图切换
- `#homeView`（全年月份视图，默认）
- `#monthView`（单个月份详情二级页）
- 通过 `.view.active` 控制显示，互相排斥。
- 顶部「返回」按钮 `homeBackBtn` 仅在 monthView 时显示。

### 全年月份视图 homeView
- 顶部 `.summary-grid`：5 个汇总卡（差旅总数 / 总费用 / 已报销 / 未报销 / 请假天数）。
- `.year-bar`：年份切换（左右箭头，范围 2000–2050）。
- `.months-grid`：12 个月卡片，**2 列(手机)/3 列(平板≥640)/4 列(桌面≥1024)**。每月卡片显示当月日历小网格 + 底部统计。

### 月份详情二级页 monthView
- `#monthCalendar`：当月考勤大日历（7 列，含上月/下月溢出灰显）。
- `.legend-bar`：四个图例（差旅 / 已报销 / 未报销 / 请假）。
- `.filter-bar`：状态筛选下拉 + 搜索框 + 结果计数。
- `#monthTravelList` / `#monthLeaveList`：当月差旅 / 请假记录卡。

### 弹窗（Modal / Sheet）
`#actionModal`（操作选择）、`#travelModal`（差旅表单）、`#leaveModal`（请假表单）、
`#datePickerModal`（**玻璃拟态日期区间选择器**）、`#dayModal`（当天记录）、`#syncModal`（云端同步）。
通用控制：`showModal(id)` / `hideModal(id)` / `closeModal(e, id)`（点击遮罩关闭）。

## 五、视觉 / 设计规范（改样式前必读）

### CSS 变量（`:root`）
- 玻璃：`--glass-bg: rgba(255,255,255,0.68)`、`--glass-border`、`--glass-shadow`
- 文字：`--text-primary:#111827`、`--text-secondary`、`--text-tertiary`
- 强调：`--accent:#111827`
- 状态色（低饱和）：
  - 差旅圆环 `--ring-travel:#93c5fd`，请假圆环 `--ring-leave:#c4b5fd`
  - 报销内点 `--dot-reimbursed:#86efac`（已报销）、`--dot-unreimbursed:#fdba74`（未报销）、`--dot-mixed:#9ca3af`（混合）
- 圆角：`--radius-sm/md/lg/xl`（12/18/24/32px）

### 玻璃拟态核心写法
```css
backdrop-filter: blur(12px) saturate(160%);  /* 同步写 -webkit- 前缀 */
border: 1px solid var(--glass-border);
box-shadow: inset 0 1px 0 rgba(255,255,255,.6), 0 8px 32px rgba(15,23,42,.07);
```
`.glass-card` 是可复用基类（带 `will-change: transform` 提升滑动性能）。

### 当前日期「浮起」效果（已定型，勿改回黑框）
```css
.month-day.today / .detail-day.today {
  background: transparent;
  font-weight: 700;
  font-size: 13px (主页) / 16px (二级页);   /* 略大于普通字 */
  text-shadow:
    0 1px 2px rgba(255,255,255,.9),        /* 白色高光 */
    0 2px 5px rgba(17,24,39,.18),          /* 柔和投影 */
    0 -1px 1px rgba(17,24,39,.06);         /* 极淡上投影 */
}
```

### 日历标记：外圈圆环 + 内部圆点（核心识别逻辑）
- 单日若有差旅：`.day-marker.travel`（蓝色 `border` 圆环）+ 内部 `.day-marker-dot` 按报销状态着色（绿/橙/灰）。
- 单日若有请假：`.day-marker.leave`（紫色圆环）+ 紫色内点。
- 同一天两者都有 → 并排两个圆环。
- 多日区间：区间内每一天都渲染相同标记（`isDateInRange` 判断）。
- 主页卡片用小标记（9px），二级页用大标记（11px）。

### 响应式断点
- `≤380px`：超小屏紧凑；`≥640px`：月份 3 列；`≥768px`：容器限宽 1000px、细节放大；`≥1024px`：月份 4 列、限宽 1180px；`≥1440px`：限宽 1360px；`landscape & max-height:600px`：横屏压缩。
- 底部导航 `.bottom-nav` 固定，`body` 已留 `padding-bottom:120px` 防止遮挡。

## 八、已知注意点 / 待办
- 多日区间的金额目前是「每条记录单笔金额」，未做区间内按天拆分；如需按天均摊可在 `saveTravel` / `renderMonthRecords` 处扩展。
- 日期选择器 `.in-range` 为圆形高亮（非连续条带），若想要连续横条需重构 grid 为行级高亮。
- 同步链接含全量数据，记录很多时 URL 会很长，极端情况建议改用 JSON 文件备份。
