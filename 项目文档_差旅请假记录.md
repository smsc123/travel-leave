## 2. 技术栈与部署

- **技术**：原生 HTML + 内联 CSS + 原生 JS，仅 CDN 引入 `lz-string@1.5.0`（压缩同步链接）。
- **存储**：浏览器 `localStorage`，键 `travelLeaveRecords`；记录**不上传** GitHub，数据私密。
- **部署**：GitHub Pages，从 `gh-pages` 分支根目录读取 `index.html`。
- **仓库分支**：
  - `main`：源码（含 `index.html` + 本文档）
  - `gh-pages`：部署用，仅 `index.html`
- **更新线上步骤**（沙箱内由模型执行）：
  ```bash
  cd /workspace
  git add index.html
  git commit -m "update"
  git push origin main
  # 同步到 gh-pages（orphan 分支，只放 index.html）
  git checkout --orphan gh-pages
  git rm -rf --cached . ; rm -f 项目文档_差旅请假记录.md
  git add index.html
  git commit -m "deploy"
  git push origin gh-pages
  git checkout -f main
  ```

## 3. 核心数据结构（改数据逻辑前必读）

`localStorage["travelLeaveRecords"]` = `{ travel:[...], leave:[...] }`

**travel[] 字段**：`id`(tr_时间戳) / `person`(必填) / `reason`(必填) / `location` / `amount`(number) / `startDate`(YYYY-MM-DD) / `endDate` / `reimbursed`(bool) / `note` / `createdAt`

**leave[] 字段**：`id`(lv_时间戳) / `type`(事假/病假/年假/调休/其他) / `startDate` / `endDate` / `note` / `createdAt`

**兼容**：旧版单日期 `date` 字段在 `normalizeRecord(record, type)` 自动转 `startDate`/`endDate`。

- **两个主视图**（互斥，`.view.active` 控制）：`#homeView`（全年 12 月卡片）/ `#monthView`（单月详情二级页）

- 通用：`showModal(id)` / `hideModal(id)` / `closeModal(e,id)`

## 6. 视觉规范（改样式前必读）

**CSS 变量** `:root`：`--glass-bg`(白0.68) / `--text-primary:#111827` / `--accent:#111827` /
`--ring-travel:#93c5fd`(差旅环) / `--ring-leave:#c4b5fd`(请假环) /
`--dot-reimbursed:#86efac`(已报销点) / `--dot-unreimbursed:#fdba74`(未报销点) / `--dot-mixed:#9ca3af`(混合)

**玻璃拟态核心**：
```css
backdrop-filter: blur(12px) saturate(160%);   /* 同步 -webkit- 前缀 */
border: 1px solid var(--glass-border);
box-shadow: inset 0 1px 0 rgba(255,255,255,.6), 0 8px 32px rgba(15,23,42,.07);
```
`.glass-card` 是可复用基类（带 `will-change:transform`）。

**当前日期「浮起」效果**（已定型，勿改黑框）：
```css
.month-day.today / .detail-day.today {
  background: transparent; font-weight: 700;
  font-size: 13px(主页)/16px(二级页);
  text-shadow: 0 1px 2px rgba(255,255,255,.9), 0 2px 5px rgba(17,24,39,.18), 0 -1px 1px rgba(17,24,39,.06);
}
```

**日历标记**：外圈圆环(`.day-marker.travel/.leave`) + 内部圆点(`.day-marker-dot` 按报销状态着色)。同日两者皆有→并排两圆环。多日区间每一天都渲染。

**响应式断点**：`≤380px` 紧凑 / `≥640px` 月份3列 / `≥768px` 限宽1000+放大 / `≥1024px` 月份4列限宽1180 / `≥1440px` 限宽1360 / `landscape&max-height:600px` 横屏压缩。`body` 留 `padding-bottom:120px` 防底部导航遮挡。

## 9. 已知待办 / 注意

- 多日区间金额是「单条记录单笔金额」，未按天拆分；如需均摊在 `saveTravel`/`renderMonthRecords` 扩展。
- 日期选择器 `.in-range` 为圆形高亮（非连续横条）；要连续条带需重构 grid 为行级高亮。
- 同步链接含全量数据，记录极多时 URL 很长，建议改用 JSON 文件备份。
- **安全**：用于部署的 GitHub Token 应已在 GitHub 撤销（Settings → Developer settings → PAT → Delete）。仓库 Public 仅网页可访问，数据存本地不泄露。
